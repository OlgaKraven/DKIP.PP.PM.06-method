1.1. Основы информационных систем и сопровождения

Раздел посвящён базовым понятиям об информационных системах, их жизненном цикле, задачах сопровождения и архитектуре. Материал служит основой для понимания последующих тем по разработке и обслуживанию Web API.

1.1.1. Понятие информационной системы

Информационная система (ИС) — это совокупность программных, технических, организационных и информационных компонентов, предназначенных для сбора, хранения, обработки и предоставления данных пользователям для решения их задач.

Информационная система обеспечивает автоматизацию процессов, повышение точности и скорости обработки данных, снижение затрат и повышение эффективности деятельности предприятия.

Компоненты информационной системы

  1. Аппаратное обеспечение (hardware)
    Серверы, рабочие станции, сетевое оборудование, устройства хранения.
  2. Программное обеспечение (software)
    • системное ПО (ОС, СУБД);
    • прикладные программы;
    • сервисы и модули ИС.
  3. Данные
    Структурированные и неструктурированные сведения, которые система хранит и обрабатывает.
  4. Пользователи
    Конечные пользователи, администраторы, разработчики.
  5. Процедуры и регламенты
    Описание процессов, правил взаимодействия, политики безопасности, инструкции эксплуатации.

1.1.2. Жизненный цикл информационной системы

Жизненный цикл ИС включает полный путь — от проектирования и создания до модернизации и вывода из эксплуатации.

Основные этапы жизненного цикла

  1. Разработка
    • анализ требований;
    • проектирование архитектуры;
    • программирование модулей;
    • тестирование;
    • подготовка документации.
  2. Внедрение
    • установка и настройка ПО;
    • миграция данных;
    • обучение пользователей;
    • пилотная эксплуатация.
  3. Сопровождение
    • поддержание работоспособности системы после запуска;
    • обновления, исправления, мониторинг, оптимизация.
  4. Модернизация
    • расширение функционала;
    • оптимизация производительности;
    • переход на новую платформу или архитектуру;
    • улучшение интерфейсов.

Жизненный цикл может быть непрерывным (DevOps, CI/CD), что обеспечивает постоянное развитие системы.

1.1.3. Основные задачи сопровождения информационных систем

Сопровождение ИС — это комплекс мероприятий, направленных на обеспечение стабильной, безопасной и эффективной работы системы после её внедрения.

  1. Развертывание
    • подготовка инфраструктуры;
    • установка сервисов, зависимостей, БД;
    • конфигурация окружений (prod/test/dev);
    • автоматизация развертывания (Docker, CI/CD).
  2. Обновление версии
    • установка патчей и исправлений ошибок;
    • миграция схемы базы данных;
    • проверка совместимости модулей;
    • откат версий при необходимости (rollback).
  3. Мониторинг работоспособности
    Контроль технических параметров:
    • нагрузка на CPU, RAM и сеть;
    • время отклика API;
    • состояние сервисов;
    • журнал ошибок.

    Используемые инструменты: Prometheus, Grafana, Zabbix, ELK.

  4. Логирование
    • сбор логов приложений и системных событий;
    • анализ ошибок и предупреждений;
    • настройка централизованного логирования;
    • обеспечение трассировки запросов.
  5. Резервное копирование
    • создание бекапов БД и файлов;
    • проверка восстановления;
    • выбор стратегии (полное, инкрементальное, дифференциальное копирование);
    • хранение копий в безопасных хранилищах.
  6. Устранение инцидентов
    • обработка обращений пользователей;
    • анализ первопричин (root cause analysis);
    • устранение сбоев в работе модулей;
    • декомпозиция ошибок на задачи для разработчиков.
Важно

Сопровождение является критическим этапом жизненного цикла ИС, обеспечивающим непрерывность бизнеса и надёжность работы системы.

1.1.4. Понятие архитектуры информационной системы

Архитектура ИС — это логическая структура системы, определяющая, как её компоненты взаимодействуют, где хранятся данные и как пользователи получают доступ к функциям.

Основные архитектурные модели

1) Клиент–серверная архитектура

  • Клиент отправляет запросы.
  • Сервер обрабатывает запросы, выполняет бизнес-логику и работает с базой данных.
  • Может быть двух- и трёхзвенной (client → server → database).

Плюсы: масштабируемость, централизованное управление.
Минусы: необходимость постоянного соединения.

2) Веб-сервисы

Используются для обмена данными между системами через интернет.

Типы:

  • REST (обычно через JSON);
  • SOAP (через XML);
  • gRPC (оптимизированный бинарный формат).

Веб-сервисы обеспечивают интеграцию модулей и микросервисов.

3) API (Application Programming Interface)

API — это интерфейс, через который другие приложения могут обращаться к функционалу системы.

Основные виды:

  • REST API — самый распространённый способ;
  • GraphQL API — гибкий запрос данных;
  • WebSocket API — обмен данными в реальном времени.

API является ключевым элементом современной архитектуры, позволяющим интегрировать системы и строить микросервисы.
