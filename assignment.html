2. Выполнение задания

В данном разделе описывается состав задания для производствееной практики, ожидаемый результат и базовая тестовая предметная область, на основе которой студенты реализуют информационную систему.

2.1. Описание задания

2.1.1. Описание задачи

Студенту предлагается выполнить следующие шаги:

  • Разработать информационную систему на языке C# с использованием ASP.NET Core Web API.
  • Реализовать базовый CRUD-функционал для выбранной предметной области (например, задачи, клиенты, товары, заявки).
  • Контейнеризировать приложение с помощью Docker.
  • Создать Dockerfile с использованием многоэтапной (multi-stage) сборки.
  • Собрать собственный Docker-образ и развернуть приложение в контейнере.
  • При необходимости подключить базу данных через docker-compose.
  • Выполнить обновление версии приложения в процессе сопровождения (например, выпустить версию v2, внести изменения и повторно развернуть).
  • Продемонстрировать работу с логами контейнеров (docker logs).
  • Реализовать и проверить работоспособность системы через health-check (endpoint /health, HEALTHCHECK в Dockerfile).
  • Выполнить базовые процедуры резервного копирования и восстановления данных (если используется база данных).
  • Подготовить краткую документацию:
    • описание системы и предметной области,
    • инструкцию запуска в Docker и через docker-compose,
    • порядок обновления (release v1 → v2),
    • примеры обращений к API (эндпоинты, запросы, ответы).
Результат

Итогом выполнения задания должен быть рабочий Web API-сервис, развёрнутый в контейнере, с реализованными CRUD-операциями, базовыми процедурами сопровождения и минимальным комплектом документации.

2.1.2. Тестовая предметная область

«Система управления заявками на ремонт и обслуживание техники в колледже»

Информационная система предназначена для учёта заявок на ремонт и обслуживание компьютерной и офисной техники в учебном заведении. Система должна позволять регистрировать, отслеживать и закрывать заявки, а также хранить информацию об оборудовании и исполнителях.

Основные сущности (под CRUD)

1. Equipment (Оборудование)

Хранит информацию о единице техники.

  • id — идентификатор;
  • inventoryNumber — инвентарный номер;
  • type — тип (ПК, принтер, проектор и др.);
  • room — кабинет/аудитория;
  • state — состояние (исправно, требует ремонта, списано и т.п.).
2. Request (Заявка)

Отражает процесс обращения по поводу поломки или обслуживания.

  • id — идентификатор заявки;
  • requester — заявитель (ФИО или группа);
  • equipmentId — ссылка на оборудование (внешний ключ к Equipment);
  • description — описание проблемы;
  • priority — приоритет (низкий, средний, высокий);
  • status — статус (новая / в работе / выполнена / отменена);
  • assigneeId — исполнитель (ссылка на User);
  • createdAt — дата и время создания заявки;
  • closedAt — дата и время закрытия заявки (опционально).
3. User (Пользователь / исполнитель)

Представляет участников процесса обработки заявок.

  • id — идентификатор пользователя;
  • fullName — ФИО;
  • role — роль (оператор, техник, администратор);
  • contacts — контактные данные (телефон, e-mail).
4. Comment (Комментарий к заявке, опционально)

Позволяет фиксировать ход выполнения работы по заявке.

  • id — идентификатор комментария;
  • requestId — ссылка на заявку;
  • authorId — автор комментария (ссылка на User);
  • text — текст комментария;
  • createdAt — время добавления комментария.

Рекомендуемый минимальный набор операций

Сущность Операции
Equipment Создание, просмотр списка, просмотр одного, обновление, удаление
Request Создание заявки, изменение статуса, назначение исполнителя, просмотр истории заявок
User Создание пользователей, назначение ролей, просмотр списка исполнителей
Comment (опционально) Добавление комментария к заявке, просмотр списка комментариев
Замечание

При желании студент может расширить выбранную из списка предметную область: добавить историю изменений статуса, файлы-вложения, категории оборудования или дополнительные роли пользователей. Важно документировать все расширения в описании системы и API.
